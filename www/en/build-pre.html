Preparing for the build

A clean start

If you have already done a NuTyX build before, you must start by removing the lfs user, the lfs group and the lfs user's home directory

So if you need to do this, remove all existing stuff:

userdel lfs
rm -r /home/lfs

Normally removing the user removes the group as well. If you are not sure about this, you can remove the group explicitly; it won't do any harm.

groupdel lfs

Partition preparation

To be able to start the first pass without major trouble, let's see what has to be done. It's quite straigtforward.

Normally when you build a Linux distribution, you create a partition to put it on. This is not actually essential: it is quite possible to build the distribution in a folder and then copy it onto a USB key, or use the rsync command to transfer it directly onto the destination disk or partition.

If you do decide to use a dedicated partition, you must now format it with the file system of your choice, then mount it on the folder you have defined by the LFS variable.

In this guide, we will use the same folder throughout:

/mnt/lfs
Once your partition is formatted, create the folder and mount your partition on it. In my example, I shall be using the second partition of the first disk. Obviously you should adjust these commands to reflect your own choice. 
mkdir /mnt/lfs
mount /dev/sda2 /mnt/lfs

Remember always to mount this partition before starting pass 1 or 2. Once the partition is mounted, we can start the first pass.
